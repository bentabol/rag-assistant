--- PROYECTO 4: RAG Assistant (junio 2026) ---
Sistema de Retrieval-Augmented Generation de producción.

¿Qué es RAG? Es una técnica que combina búsqueda semántica con generación de texto por LLMs.
En lugar de pedirle al modelo que recuerde información (y que se la invente), el sistema primero busca
los fragmentos relevantes en una base de documentos real, y luego le pide al modelo que responda
SOLO basándose en esos fragmentos, citando la fuente.

Problema que resuelve: Los LLMs tienen conocimiento congelado hasta su fecha de entrenamiento
y no conocen documentos privados. Si le preguntas a ChatGPT sobre un contrato tuyo, se lo inventa.
RAG resuelve esto de forma eficiente y verificable.

Pipeline técnico completo:
1. INGESTA: El documento (PDF, texto, URL web) se carga y trocea en fragmentos de ~800 caracteres
   con solapamiento de 120 caracteres para preservar contexto entre fragmentos.
2. EMBEDDINGS: Cada fragmento se convierte en un vector numérico de 384 dimensiones usando
   el modelo sentence-transformers/all-MiniLM-L6-v2 (local, gratuito, rápido).
   Un embedding captura el significado semántico: frases con significado similar tienen vectores similares.
3. VECTOR STORE: Los vectores se almacenan en ChromaDB (base de datos vectorial persistente en disco)
   con métrica de similitud coseno (cosine similarity).
4. RETRIEVAL: Cuando el usuario hace una pregunta, se convierte también en un vector y se buscan
   los K fragmentos más similares en ChromaDB (top-K retrieval).
5. GENERACIÓN: Solo esos K fragmentos relevantes se envían al LLM (Llama 3.3 70B via Groq API, gratis)
   con un prompt que le obliga a citar fuentes y admitir incertidumbre si no tiene información suficiente.

Stack técnico:
- LangChain: orquestación del pipeline.
- sentence-transformers (all-MiniLM-L6-v2): embeddings locales y gratuitos.
- ChromaDB: base de datos vectorial persistente.
- Groq API (Llama 3.3 70B): LLM gratuito, 6000 peticiones/día.
- Gradio: interfaz web interactiva con tres pestañas (Ingest, Ask, Stats).
- PyPDF: extracción de texto de PDFs.
- BeautifulSoup4: extracción de texto de páginas web.

Características destacadas:
- Soporte multi-formato: PDF, TXT, Markdown, URLs web.
- Chunking inteligente con RecursiveTextSplitter.
- Embeddings locales: sin coste de API para la indexación.
- ChromaDB persistente: los documentos sobreviven reinicios del servidor.
- LLM constrained: Claude o Llama forzado a citar fuentes, no puede alucinar.
- Interfaz Gradio limpia con chat, upload y estadísticas del índice.
- CLI scriptable para automatización.

Por qué es valioso para FAANG/empresas grandes:
- RAG es el patrón más usado en IA empresarial en 2026 (AWS Bedrock, Azure AI Search, Google Vertex AI lo usan por debajo).
- Permite a empresas hacer preguntas sobre su documentación interna, contratos, bases de conocimiento.
- Escala a millones de documentos sin aumentar el coste del LLM.
- Elimina alucinaciones al forzar respuestas basadas en evidencia.

--- RAG Assistant (para explicar a no técnicos) ---

Imagina que tienes 1000 documentos PDF de tu empresa y quieres hacerle preguntas a un asistente de IA sobre ellos.
El problema es que la IA (como ChatGPT) no sabe nada de tus documentos — solo sabe lo que aprendió antes de su entrenamiento.
Si le preguntas, se inventa las respuestas.

El RAG Assistant resuelve esto así:
1. Primero "lee" todos tus documentos y los organiza de forma inteligente en una base de datos.
2. Cuando haces una pregunta, busca en esa base de datos los fragmentos más relevantes.
3. Le pasa esos fragmentos al asistente de IA y le dice: "responde SOLO con esto, y cita de dónde sacas cada dato".
4. El asistente responde con información real, diciendo de qué documento y página viene cada afirmación.

Resultado: un asistente que puede responder preguntas sobre tus propios documentos, sin inventarse nada.
Caso de uso empresa: "¿Qué dice nuestro contrato con el proveedor X sobre penalizaciones?" → respuesta exacta con cita de página.

--- TFM — Clasificación de imágenes (para explicar a no técnicos) ---

Desarrollé un sistema que aprende a reconocer qué hay en una foto.
En lugar de construir el sistema desde cero (que requeriría millones de fotos), usé modelos que ya habían aprendido
a reconocer miles de objetos (entrenados en ImageNet con 14 millones de imágenes).
